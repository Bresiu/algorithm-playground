# Algorithm Playground

Personal playground for learning algorithms, data structures and problem solving using Kotlin.

The goal of this repository is to become better at solving problems, understanding algorithmic patterns and writing clean, efficient solutions.

This is not a collection of copied LeetCode solutions. It is a long-term learning project that documents my progress, notes and different approaches to solving problems.

## Technology

* Kotlin
* Gradle
* JUnit 5

## Learning Principles

* Understand the problem before writing code.
* Find a simple solution first.
* Improve it step by step.
* Focus on reasoning instead of memorizing solutions.
* Learn patterns that can be reused in many problems.
* Write clean and readable Kotlin.
* Always analyze time and space complexity.

## Workflow

Each problem follows the same process:

1. Understand the requirements.
2. Discuss possible approaches.
3. Identify the appropriate algorithm or data structure.
4. Write pseudocode.
5. Implement the solution in Kotlin.
6. Write unit tests.
7. Analyze:

    * Time complexity
    * Space complexity
    * Edge cases
    * Possible optimizations
8. Summarize the pattern and related problems.
